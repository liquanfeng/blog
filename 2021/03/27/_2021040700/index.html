感谢这位知乎作者幼儿园级解释，最后总结下以后忘了回来看比较快回想起来 假设字符串为s，

  1. 首先像下面插入字符串预处理一下保证处理字符串个数为奇数，^$为了保证首尾不等防止计算溢出
    ^a#b#c$
  2. 遍历字符串s的每个字符s[i]，同时更新已知最靠右的回文子串两个属性：中心下标C和最右下标R。初始R=C=0
  3. 对于第i次迭代，有三种状态
    • R<=i
      老实以C为中心，以i起点拓展回文子串并更新R
    • R>i and R-i>P[i_mirror]（i_mirror是以C为中心R的对称点）
      这个利用到回文子串的特性，在以P[i_mirror]为中心的回文子串的长度没有超过已知最右回文子串范围内的情况下，p[i]其实就是P[i_mirror]
    • R>i and R-i<=P[i_mirror]
      这个就是上面第二种情况里的例外，以P[i_mirror]为中心的回文子串的长度超过已知最长回文子串范围的情况，未知范围可能可以再拓展回文子串，所以开始拓展

所以一个原始易懂的版本来了

def manacher(s):
    P = [0] * len(s)
    R, C = 0, 0
    for i in range(1, len(s) - 1):
        i_mirror = 2 * C - i
        if R <= i:
            # 新起点
            pass
        else: # R > i
            if R - i > P[i_mirror]:
                # 包含，直接返回
                P[i] = P[i_mirror]
                continue
            else:
                # 相交，尝试从R处开始扩展。
                # 因为下一个比较的是s[i + P[i] + 1]，由于P[i]=R-i,所以比较的就是s[R+1]
                P[i] = R - i
        # 尝试匹配
        while s[i - P[i] - 1] == s[i + P[i] + 1]:
            P[i] += 1
        # 更新R，C
        if i + P[i] > R:
            R, C = i + P[i], i
    return P

时间复杂度O(n)，这个是因为里面的while循环在整个计算的过程中，R是从0开始单向增大的，是O(n)，加上一开始的for循环总共O(2n)

最后是容易记住的版本

def manacher(s):
    T = '#'.join('^{}$'.format(s))
    P = [0] * len(T)
    R, C = 0, 0
    for i in range(1,len(T) - 1):
        if i < R:
            P[i] = min(P[2 * C - i], R - i)
        while T[i+(P[i]+1)] == T[i-(P[i]+1)]:
            P[i] += 1
        if i + P[i] > R:
            R, C = i + P[i], i
    return P
