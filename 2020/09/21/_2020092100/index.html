# Polly: Symbol `fprintf' causes overflow in R_X86_64_PC32 relocation

今天按polly文档跑了一下，遇到这样一个问题

$ time ./matmul.polly.interchanged+tiled.out
  ./matmul.polly.interchanged+tiled.out: Symbol `fprintf' causes overflow in R_X86_64_PC32 relocation`

根据stackoverflow，gcc编译加上-no-pie

$ gcc matmul.polly.interchanged+tiled.s -o matmul.polly.interchanged+tiled.out -no-pie
$ time ./matmul.polly.interchanged+tiled.out 
  real	0m0.384s
  user	0m0.364s
  sys	0m0.020s
